# Daily Briefing — 日报生成主技能

## 功能概览

| 类型 | 内容 | 适用人群 | 子 Skill |
|------|------|----------|----------|
| 通用日报 | 地缘政治、金融市场、科技AI、中国政策等多领域 | 普通读者 | `skills/daily-core/` |
| 投资日报 | 全球融资动态、赛道分析、趋势洞察 | VC/PE投资人 | `skills/vc-daily/` |
| 股票日报 | 自选股监控、个股新闻、行情数据 | 股票投资者 | `skills/stock-daily/` |
| AI效率日报 | GitHub/HuggingFace/官方博客等AI工具动态 | 开发者/超级个体 | `skills/ai-daily/` |

输出格式：**PDF** / **飞书文档** / **飞书消息卡片（消息流）**

---

## Step 1：读取或初始化用户配置

检查工作区是否存在 `workspace/config/daily-briefing-config.txt`：

```
类型: 通用          # 通用 / 投资 / 股票 / AI效率
关注: 科技,金融,中国  # 关注领域关键词（通用日报使用）
自选股: 600519,AAPL  # 股票代码（股票日报使用）
输出: PDF           # PDF / 飞书文档 / 消息流
语言: 中文           # 中文 / 英文
```

**首次使用（无配置文件）**：引导用户完成配置：
1. 询问日报类型（通用 / 投资 / 股票 / AI效率）
2. 询问关注领域或自选股列表
3. 询问输出格式
4. 将配置写入 `workspace/config/daily-briefing-config.txt`

**已有配置**：直接读取，并告知用户当前配置，询问是否沿用。

---

## Step 2：路由分发

根据配置中的`类型`字段，调用对应子 Skill：

```
类型 = 通用     → 读取 skills/daily-core/SKILL.md
类型 = 投资     → 读取 skills/vc-daily/SKILL.md
类型 = 股票     → 读取 skills/stock-daily/SKILL.md
类型 = AI效率   → 读取 skills/ai-daily/SKILL.md
```

读取对应子 Skill 后，按其指令执行内容采集与生成。

---

## Step 3：格式转换与输出

子 Skill 返回内容后，根据`输出`字段处理：

### PDF 输出
1. 读取对应模板（`templates/` 目录）
2. 将内容填充至 HTML 模板
3. 使用 Playwright 将 HTML 渲染为 PDF
4. 保存为 `outputs/daily_report_YYYYMMDD.pdf`
5. 发送文件给用户

### 飞书文档输出
1. 将结构化内容转换为飞书文档块（Block）格式
2. 调用飞书文档 API 创建文档
3. 返回文档链接

### 消息流输出
1. 将内容格式化为 Markdown 文本
2. 直接在对话中发送（支持分段推送）

---

## 注意事项

- **不要捏造数据**：所有新闻、行情数据必须来自实际搜索结果
- **来源必须附链接**：每条新闻/资讯需附可点击的原始链接
- **日期准确**：明确标注内容所属日期，避免旧闻混入
- **网络失败处理**：若搜索受限，在报告顶部注明"部分数据获取受限"，不编造内容

---

## 扩展新类型

添加新日报类型步骤：
1. 在 `skills/` 下创建新目录，如 `skills/research-daily/`
2. 编写 `SKILL.md`，定义内容采集和输出格式
3. 在配置文件中设置 `类型: 学术`
4. 在本文件 Step 2 路由表中添加对应条目
